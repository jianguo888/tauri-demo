# 🌤 初识 Tauri + HarmonyOS — 这个天气应用到底是怎么工作的？

> **阅读对象**：编程小白，对 Tauri 和鸿蒙开发零基础  
> **目标**：看懂这个项目的全貌，理解"前端 + 后端 + 鸿蒙"这三者是怎么拼在一起的

---

## 1. 这个项目是什么？

这是一个 **天气查询 App**，它运行在 **鸿蒙（HarmonyOS）手机** 上。

你在手机上打开它 → 选择省份和城市 → 点击"查询天气"→ 立刻看到温度、湿度、空气质量、未来 24 小时预报……

而它的"真身"是一个 **Tauri 项目**。什么是 Tauri？

> **Tauri** 是一个"跨平台应用框架"。你可以用 **Web 技术（HTML/CSS/JavaScript）写界面**，用 **Rust 语言写底层逻辑**，然后打包成手机 App 或桌面软件。

简单说：**把网页变成真正的 App**。

---

## 2. 项目长什么样？（目录结构）

```
tauri-demo/                     ← 项目根目录
├── src/                        ← 👈 前端代码（Vue 框架）
│   ├── App.vue                 ←    主页面（天气查询界面）
│   ├── main.ts                 ←    Vue 入口文件
│   └── vite-env.d.ts           ←    类型声明
├── src-tauri/                  ← 👈 后端代码（Rust 语言）
│   ├── src/
│   │   ├── lib.rs              ←    核心逻辑（天气 API 调用）
│   │   └── main.rs             ←    程序入口
│   ├── Cargo.toml              ←    Rust 依赖清单
│   ├── tauri.conf.json         ←    Tauri 配置
│   ├── capabilities/           ←    权限声明
│   └── gen/ohos/               ←    👈 生成的鸿蒙项目
│       └── entry/              ←        鸿蒙模块（用 DevEcoStudio 打开）
├── package.json                ← 前端依赖清单
├── index.html                  ← 网页入口
├── .env                        ← 环境变量（API Token）
└── pnpm-workspace.yaml         ← pnpm 配置
```

你看，这个项目本质上只有 **三样东西**：

| 目录 | 语言 | 负责什么 |
|------|------|---------|
| `src/` | TypeScript / Vue | **界面** — 你看到的按钮、卡片、下拉框 |
| `src-tauri/src/` | Rust | **逻辑** — 调用天气 API、处理数据 |
| `src-tauri/gen/ohos/` | ArkTS / 鸿蒙 | **打包壳** — 让 App 能在鸿蒙上运行 |

## 3. 一次"查询天气"的完整旅程

你在手机上选"北京"→ 点"查询天气"，背后发生了什么？

```
你点击按钮
     │
     ▼
┌─────────────────────────────────────────┐
│  Vue 前端 (src/App.vue)                 │
│  调用 invoke("get_weather", {...})       │
│  ─────────────────────────────────────  │
│  把"北京"通过 Tauri 桥 传给 Rust 后端     │
└──────────┬──────────────────────────────┘
           │  invoke() 跨语言调用
           ▼
┌─────────────────────────────────────────┐
│  Rust 后端 (src-tauri/src/lib.rs)       │
│  收到请求 → 组装参数 →                   │
│  POST https://v3.alapi.cn/api/tianqi     │
│  拿到 JSON 数据 → 原样返回                │
└──────────┬──────────────────────────────┘
           │  返回 JSON 字符串
           ▼
┌─────────────────────────────────────────┐
│  Vue 前端 (src/App.vue)                 │
│  解析 JSON → 渲染成漂亮的天气卡片         │
│  显示温度☀️ 湿度💧 空气质量🌿 ...          │
└─────────────────────────────────────────┘
```

**核心理解**：
- 前端 **只负责界面好看**，它不直接调天气 API
- 后端 **Rust** 负责真正去互联网上拉数据
- 前后端通过 `invoke()` 这座"桥"通信 —— 这是 Tauri 最核心的机制

## 4. 为什么用 Tauri 做鸿蒙 App？

传统上，开发鸿蒙 App 需要用 **ArkTS（鸿蒙自己的语言）** 和 **DevEco Studio**。但 Tauri 让你：

| 传统方式 | Tauri 方式 |
|----------|-----------|
| 学 ArkTS | ✅ 用你熟悉的 Vue / React |
| 鸿蒙独享代码 | ✅ 一套代码也能编译到 Windows / macOS |
| UI 和逻辑混在一起 | ✅ 界面（Vue）+ 逻辑（Rust）分离 |
| 网络请求用鸿蒙 API | ✅ 用 Rust 的 reqwest 库，跨平台通用 |

**代价**：需要一个额外的"壳"把 Web 界面包装成原生 App —— 这就是 `src-tauri/gen/ohos/` 里的鸿蒙项目，它是自动生成的，你基本不用碰它。

---

## 5. 小白最容易困惑的几个概念

### ❓ "这不就是个网页吗？怎么就成了 App？"

你的前端代码 (`src/`) 最终被 **Vite** 打包成静态文件（HTML + JS + CSS），然后 Tauri 把它们"塞进"一个鸿蒙原生窗口里。那个窗口是一个 **WebView**（类似浏览器内核），它把你的网页渲染出来，但**没有浏览器的地址栏**，看起来就是原生 App。

### ❓ "Rust 是什么？我前端不用学吧？"

Rust 是一种系统编程语言，以**安全、高性能**著称。在这个项目里：
- 你**不需要**精通 Rust 也能看懂
- 它只做一件事：**调外部 API 取数据**
- 如果你只想改界面，完全不用碰 Rust

### ❓ "gen/ohos 目录是怎么回事？"

它是构建过程中**自动生成**的鸿蒙工程。每次你运行 `cargo tauri ohos build`，Tauri 都会重新生成它。你可以把它理解为："Tauri 帮我把我的代码翻译成了鸿蒙能懂的项目"。
