# 🚀 实战案例 7：从代码到设备 — 自动化编译与部署

> **目标**：摆脱 DevEco Studio 的 GUI 依赖，一行命令完成编译 → 安装 → 启动的全流程  
> **适用场景**：持续集成、远程开发、快速迭代调试

---

## 一、为什么需要自动化？

### 现状痛点

日常开发中，每次修改代码后部署到鸿蒙设备需要重复以下步骤：

| 步骤 | 操作 | 耗时 |
|------|------|------|
| 1 | 打开 DevEco Studio | ~10s |
| 2 | 等待项目加载 | ~5s |
| 3 | 点击 Build → Build HAP(s) | ~30s |
| 4 | 点击 Run → 选择设备 | ~10s |
| 5 | 等待安装和启动 | ~10s |

每次部署 **~65 秒**，而且手不能离开鼠标。一天 20 次就是 20 分钟。

### 自动化方案

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 代码修改  │───▶│ cargo    │───▶│ hdc      │───▶│ aa start │
│ (任何编辑器)│    │ tauri    │    │ install  │    │ 启动 App │
└──────────┘    │ ohos     │    └──────────┘    └──────────┘
                │ build    │
                └──────────┘
```

**一行搞定**：
```bash
zsh deploy.sh
```

## 二、工具链准备

### 2.1 确认 hdc 已安装

```bash
hdc list targets
```

输出示例：
```
192.168.10.6:36937
```

如果提示 `command not found`，说明 hdc 不在 PATH 中。hdc 通常位于 DevEco Studio 的 SDK 目录：

```bash
# macOS 典型路径
export PATH="$HOME/Huawei/Sdk/hmscore/3.12.0.300/hdc/mac:$PATH"

# 或找到你的 DevEco Studio 中的 hdc
find ~/ -name "hdc" -type f 2>/dev/null
```

### 2.2 确认 bundleName

鸿蒙应用的 bundle 名称定义在 `AppScope/app.json5`：

```json5
{
  "app": {
    "bundleName": "com.ranger.tauri_demo",  // ← 记住这个
    "versionCode": 1,
    "versionName": "1.0.0"
  }
}
```

### 2.3 确认 abilityName

在 `entry/src/main/module.json5` 中查看：

```json5
"abilities": [
  {
    "name": "EntryAbility",  // ← 能力名称
    "srcEntry": "./ets/entryability/EntryAbility.ets",
    // ...
  }
]
```

## 三、一键部署脚本

在项目根目录创建 `deploy.sh`：

```bash
#!/bin/bash
# deploy.sh — 一键编译 + 安装 + 启动鸿蒙应用
# 用法: ./deploy.sh [--debug]

set -e

# ============ 配置 ============
BUNDLE_NAME="com.ranger.tauri_demo"
ABILITY_NAME="EntryAbility"

# Tauri 项目路径
TAURI_DIR="src-tauri"
# HAP 输出路径（相对 TAURI_DIR）
HAP_OUTPUT="gen/ohos/entry/build/default/outputs/default"

# ============ 解析参数 ============
BUILD_MODE="release"
BUILD_FLAG=""
if [ "$1" == "--debug" ]; then
  BUILD_MODE="debug"
  BUILD_FLAG="-d"
fi

echo "========================================"
echo " 🚀 鸿蒙一键部署 ($BUILD_MODE)"
echo "========================================"

# 1. 编译 HAP
echo ""
echo "📦 Step 1/3: 编译 HAP..."
cd "$TAURI_DIR"
cargo tauri ohos build $BUILD_FLAG
cd ..

# 2. 查找 HAP 文件
echo ""
echo "📲 Step 2/3: 安装到设备..."
if [ "$BUILD_MODE" == "release" ]; then
  HAP_FILE="$TAURI_DIR/$HAP_OUTPUT/entry-default-signed.hap"
else
  HAP_FILE="$TAURI_DIR/$HAP_OUTPUT/entry-default-unsigned.hap"
fi

if [ ! -f "$HAP_FILE" ]; then
  echo "❌ 未找到 HAP 文件: $HAP_FILE"
  exit 1
fi

hdc install "$HAP_FILE"
echo "✅ 安装成功"

# 3. 启动应用
echo ""
echo "▶️  Step 3/3: 启动应用..."
hdc shell aa start -a "$ABILITY_NAME" -b "$BUNDLE_NAME"
echo "✅ 启动成功"

echo ""
echo "========================================"
echo " ✅ 部署完成！应用已在设备上运行"
echo "========================================"
```

### 使用方式

```bash
# 赋权
chmod +x deploy.sh

# 正式部署
./deploy.sh

# Debug 模式
./deploy.sh --debug
```

## 四、进阶：自动重试与错误处理

设备偶尔会断开连接，以下脚本增加了重试逻辑：

```bash
#!/bin/bash
# deploy-robust.sh — 健壮版一键部署

set -e
BUNDLE_NAME="com.ranger.tauri_demo"
ABILITY_NAME="EntryAbility"

# 编译
cd src-tauri
cargo tauri ohos build
cd ..

HAP="src-tauri/gen/ohos/entry/build/default/outputs/default/entry-default-signed.hap"

# 安装（最多重试 3 次）
for i in 1 2 3; do
  echo "📲 安装尝试 #$i"
  if hdc install "$HAP" 2>/dev/null; then
    echo "✅ 安装成功"
    break
  fi
  if [ $i -eq 3 ]; then
    echo "❌ 安装失败，请检查设备连接"
    exit 1
  fi
  echo "⏳ 等待 3 秒后重试..."
  sleep 3
done

# 启动
hdc shell aa start -a "$ABILITY_NAME" -b "$BUNDLE_NAME" && echo "✅ 启动成功"
```

## 五、进阶：集成到 `pnpm` / `npm` 脚本

将部署命令加入 `package.json`，这样 `pnpm deploy` 即可完成一切：

```json
{
  "scripts": {
    "dev": "vite",
    "build": "vue-tsc --noEmit && vite build",
    "deploy": "cd src-tauri && cargo tauri ohos build && hdc install gen/ohos/entry/build/default/outputs/default/entry-default-signed.hap && hdc shell aa start -a EntryAbility -b com.ranger.tauri_demo",
    "deploy:debug": "cd src-tauri && cargo tauri ohos build -d && hdc install gen/ohos/entry/build/default/outputs/default/entry-default-unsigned.hap && hdc shell aa start -a EntryAbility -b com.ranger.tauri_demo"
  }
}
```

使用：

```bash
pnpm deploy          # release 模式部署
pnpm deploy:debug    # debug 模式部署
```

## 七、常见问题

### Q1: `hdc list targets` 显示空

**原因**：设备未连接或授权未通过。  
**解决**：
```bash
# 检查 USB / 网络连接
hdc list targets -v

# 如果通过 TCP 连接
hdc tconn 192.168.10.6:36937
```

### Q2: `hdc install` 报 `INSTALL_PARSE_FAILED`

**原因**：HAP 签名问题或版本不匹配。  
**解决**：确保使用 release 模式构建（`-d` 构建的 unsigned HAP 需手动签名）。

### Q3: `aa start` 失败

**原因**：bundleName 或 abilityName 错误，或应用尚未完全安装。  
**解决**：
```bash
# 查看已安装的应用列表
hdc shell bm dump -a | grep "com.ranger"
```

## 八、小结

| 脚本 | 用途 | 命令 |
|------|------|------|
| `deploy.sh` | Release 一键部署 | `./deploy.sh` |
| `deploy.sh --debug` | Debug 一键部署 | `./deploy.sh --debug` |
| `pnpm deploy` | 集成到 pnpm 脚本 | `pnpm deploy` |

**核心价值**：
- 从源码到真机运行，**18 秒** vs 手工操作 **65 秒**
- 无 GUI 依赖，可在任何编辑器、CI 环境、远程终端中执行
- 自动化让「改一行代码 → 看效果」的反馈循环大幅缩短
