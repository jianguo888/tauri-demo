# 📥 实战案例 2：文件下载与图片处理 — 纯 Rust 替代的完整适配

> **目标**：在天气 App 中添加"天气海报分享"功能 — 从网络下载背景图片，用 Rust 合成含天气信息的图片  
> **核心**：演示 `reqwest` 下载文件 + `image` 库纯 Rust 模式解码 + 验证"全部纯 Rust = 零适配成本"

---

## 一、场景设计

延续收藏城市功能，增加"生成天气海报"：

```
                    ┌──────────────────────┐
                    │   🌤 北京 天气日报     │
                    │                      │
                    │   ☀️   26°            │
                    │  晴   16°~29°        │
                    │                      │
                    │   💧湿度 20%  🌬️ 11km/h│
                    │   🌅日出 04:45  🌇19:45│
                    │                      │
                    │     空气质量 优 36     │
                    │   ─────────────      │
                    │  数据来源: ALAPI      │
                    └──────────────────────┘
```

流程：
1. 前端点击"生成海报"
2. Rust 后端下载一张天空背景图（从网络）
3. Rust 后端用 `image` 库在图片上叠加天气文字
4. 返回 Base64 图片给前端展示
5. 用户可长按保存

## 二、适配步骤

### Step 1：添加依赖

```toml
# src-tauri/Cargo.toml
[dependencies]
# ... 已有 ...

# image 库：纯 Rust 编解码
# 只开我们需要的格式，避免系统库依赖
image = { version = "0.25", default-features = false, features = [
    "png",          # 输出 PNG
    "jpeg",         # 输入 JPEG（背景图）
] }

# 字体渲染（纯 Rust）
rusttype = "0.9"   # 字体解析 + 点阵渲染
```

**关键决策**：

| 库 | 默认 feature | 鸿蒙问题 | 我们的选择 |
|----|-------------|---------|-----------|
| `image` | 所有格式 | 需要 `libjpeg`、`libpng`、`libtiff`…… | `default-features = false`，只开 `png` + `jpeg` |
| `image` 解码 | 系统 libjpeg | 鸿蒙没有 | 使用 `jpeg-decoder`（纯 Rust，`image` 默认带） |
| `image` 编码 | 系统 libpng | 鸿蒙没有 | `image` 的 PNG 编码已内置纯 Rust 实现 |
| `rusttype` | - | 仅加载字体文件，无系统依赖 | 直接用 |

**为什么 `image` 的 "jpeg" feature 不需要系统 libjpeg？**

`image` 库很聪明：它提供了两种 JPEG 解码器：

- `jpeg` → 使用 `jpeg-decoder` crate（**纯 Rust** ✅）
- `jpeg_rayon` → 同上但并行加速

所以即使开 JPEG 支持，也不依赖系统 C 库。

## 三、核心代码实现

### 3.1 海报生成模块

```rust
// src-tauri/src/poster.rs
use image::{DynamicImage, RgbaImage, Rgba};
use image::codecs::png::PngEncoder;
use std::io::Cursor;

/// 天气海报数据
pub struct WeatherPoster {
    pub city: String,
    pub temp: i32,
    pub weather: String,
    pub min_temp: i32,
    pub max_temp: i32,
    pub humidity: String,
    pub wind_speed: String,
    pub sunrise: String,
    pub sunset: String,
    pub air_quality: String,
    pub air_level: String,
}

/// 生成天气海报，返回 PNG 字节数组
pub fn generate_weather_poster(
    data: &WeatherPoster,
    background: &[u8],  // 背景图字节
) -> Result<Vec<u8>, String> {
    // 1. 解码背景图
    let img = image::load_from_memory(background)
        .map_err(|e| format!("解码背景图失败: {e}"))?;

    // 2. 缩放到标准尺寸 800×1200
    let mut canvas = img.resize_exact(800, 1200, image::imageops::FilterType::Lanczos3)
        .to_rgba8();

    // 3. 叠加半透明遮罩（让文字更清晰）
    let overlay = RgbaImage::from_pixel(800, 1200, Rgba([0, 0, 0, 100]));
    image::imageops::overlay(&mut canvas, &overlay, 0, 0);

    // 4. 绘制文字（用简单的像素绘制，不用字体库）
    draw_text(&mut canvas, &format!("{} 天气日报", data.city), 40, 60, 48);
    draw_text(&mut canvas, &format!("{}°C", data.temp), 60, 160, 96);

    // 5. 天气状况 & 温度范围
    draw_text(&mut canvas, &format!(
        "{}    {}°C ~ {}°C",
        data.weather, data.min_temp, data.max_temp
    ), 60, 300, 32);

    // 6. 详细信息
    let details = vec![
        format!("💧 湿度    {}", data.humidity),
        format!("🌬️ 风速    {}", data.wind_speed),
        format!("🌅 日出    {}", data.sunrise),
        format!("🌇 日落    {}", data.sunset),
        format!("🌿 空气质量  {} ({})", data.air_quality, data.air_level),
    ];
    for (i, line) in details.iter().enumerate() {
        draw_text(&mut canvas, line, 60, 400 + i as u32 * 50, 28);
    }

    // 7. 编码为 PNG
    let mut output = Vec::new();
    let encoder = PngEncoder::new(&mut output);
    encoder
        .encode(
            &canvas,
            canvas.width(),
            canvas.height(),
            image::ExtendedColorType::Rgba8,
        )
        .map_err(|e| format!("编码 PNG 失败: {e}"))?;

    Ok(output)
}

/// 简易文字绘制（用矩形模拟像素字体，实际项目建议用 rusttype / ab_glyph）
fn draw_text(canvas: &mut RgbaImage, text: &str, x: u32, y: u32, font_size: u32) {
    // 这是一个简化实现
    // 真实项目中你会用 rusttype 或 ab_glyph 加载 .ttf 字体文件渲染
    // 这里用一个白色实心背景条 + 文字模拟
    let bar_height = font_size + 16;
    for dy in 0..bar_height {
        for dx in 0..canvas.width().min(x + text.len() as u32 * (font_size / 2 + 4)) {
            if dx >= x && dy >= y {
                if let Some(pixel) = canvas.get_pixel_mut_checked(dx, dy) {
                    // 半透明白底
                    *pixel = Rgba([
                        pixel[0].saturating_add(20),
                        pixel[1].saturating_add(20),
                        pixel[2].saturating_add(20),
                        200,
                    ]);
                }
            }
        }
    }
}
```

### 3.2 Tauri 命令

```rust
// src-tauri/src/lib.rs
mod poster;
use base64::{Engine as _, engine::general_purpose::STANDARD};

/// 生成天气海报（带网络下载背景图）
#[tauri::command]
async fn generate_poster(
    city: String,
    temp: i32,
    weather: String,
    min_temp: i32,
    max_temp: i32,
    humidity: String,
    wind_speed: String,
    sunrise: String,
    sunset: String,
    air: String,
    air_level: String,
) -> Result<String, String> {
    // 1. 下载背景图
    let bg_url = "https://images.unsplash.com/photo-1504608524841-42fe6f032b4b?w=800";
    let client = reqwest::Client::new();
    let bg_bytes = client
        .get(bg_url)
        .send()
        .await
        .map_err(|e| format!("下载背景图失败: {e}"))?
        .bytes()
        .await
        .map_err(|e| format!("读取背景图失败: {e}"))?;

    // 2. 构建海报数据
    let poster_data = poster::WeatherPoster {
        city,
        temp,
        weather,
        min_temp,
        max_temp,
        humidity,
        wind_speed,
        sunrise,
        sunset,
        air_quality: air,
        air_level,
    };

    // 3. 生成海报
    let png_bytes = poster::generate_weather_poster(&poster_data, &bg_bytes)?;

    // 4. 转 Base64 返回前端
    Ok(STANDARD.encode(&png_bytes))
}
```

注意需要添加 `base64` 依赖：

```toml
# src-tauri/Cargo.toml

```

## 五、深入理解：为什么这些库不需要适配？

```
                    ┌───────────────────────────────┐
                    │    你的 Rust 代码               │
                    │    poster.rs                   │
                    └────────┬──────────────────────┘
                             │ 调用
                    ┌────────▼──────────────────────┐
                    │  image 库                      │
                    │  ├── jpeg-decoder (纯 Rust)    │
                    │  ├── png (纯 Rust)             │
                    │  └── imageops (纯 Rust)        │
                    └────────┬──────────────────────┘
                             │ 全是 Rust 代码
                             │ 不需要系统库
                    ┌────────▼──────────────────────┐
                    │  鸿蒙 ARM64 CPU                │
                    │  Rust 编译器生成机器码           │
                    │  直接运行                       │
                    └───────────────────────────────┘
```

**每一层都是纯 Rust**，没有 FFI 调用 → 不需要目标平台提供任何系统库 → **跨平台零成本适配**。

## 六、哪些格式的 `image` 是纯 Rust 的？

从 `image` 0.25 的 features 文档：

| 格式 | 解码 feature | 是否纯 Rust | 编码 feature | 是否纯 Rust |
|------|-------------|-----------|-------------|-----------|
| PNG | `png` | ✅ `png` crate | `png` | ✅ |
| JPEG | `jpeg` | ✅ `jpeg-decoder` | `jpeg` | ✅ |
| GIF | `gif` | ✅ | ❌ | - |
| WebP | `webp` | ✅ | `webp` | ✅ |
| BMP | `bmp` | ✅ 内置 | `bmp` | ✅ 内置 |
| TIFF | `tiff` | ✅ `tiff` | ❌ | - |
| AVIF | `avif` | ✅ | ❌ | - |
| HEIF/HEIC | 实验性 | ⚠️ 部分实现 | ❌ | - |

**验证方法**：看这个格式的 crate 是否依赖 C 库。`jpeg-decoder`、`png`、`gif` 都是纯 Rust 实现。

## 七、字体渲染的选择

上面代码中的 `draw_text` 用了非常简化的实现。实际项目中你可以：

| 库 | 方法 | 鸿蒙兼容性 |
|----|------|-----------|
| `rusttype` | 加载 `.ttf` → 解析字形 → 位图渲染 | ✅ 纯 Rust |
| `ab_glyph` | `rusttype` 的进化版，更现代 | ✅ 纯 Rust |
| `imageproc` | 构建在 `image` 之上的图像处理工具集 | ✅ 含文字渲染 |
| 直接嵌入点阵字体 | 把字体转换成字节数组内嵌 | ✅ 零依赖 |

推荐组合：**`ab_glyph` + `imageproc`**，两者都是纯 Rust，且 API 更友好。

## 八、总结

```
适配鸿蒙的 Rust 库选择原则
─────────────────────────
优先选：纯 Rust 实现         ← image、serde、regex
其次选：bundled 模式         ← rusqlite、flate2
再次选：换 feature           ← reqwest (rustls-tls)
最后选：fork + patch         ← tauri、openharmony-ability
```

**本案例的关键验证**：`image` 库所有我们需要的功能都是纯 Rust 实现，**在鸿蒙上无需任何修改即可编译运行**。这说明 Rust 生态中已经有大量跨平台可用的高质量库，适配的障碍远比你想象的小。
